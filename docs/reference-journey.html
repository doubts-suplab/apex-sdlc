The APEX Reference Journey

One project — a Customer Refunds Service — walking all seven SDLC phases and seven personas. Every artifact is generated by the governed agent-harness, offline, from a one-paragraph brief. The agent proposes a decision; the harness decides what may auto-enforce.

The founding principle is enforced, not asserted. The four SUGGEST-authority phases can never auto-enforce (harness gate rule G-5), so they always route to a human — "AI generates drafts; humans approve" is a property of the authority ladder, not a coding convention.
OBSERVE < SUGGEST < ALERT < RATE_LIMIT < BLOCK │ │ │ │ never auto-enforce └──── auto-enforce only when confidence ≥ threshold ────┘ (humans approve) ALERT 0.80 RATE_LIMIT 0.85 BLOCK 0.95

Regenerate it yourself

The whole journey runs offline — no database, no API keys.

cd platform/backend python -m venv .venv && . .venv/bin/activate && pip install -e . LLM_PROVIDER=stub python -m app.demo.reference_journey # writes examples/reference-project/artifacts/**

Or serve it: GET /api/v1/journey/reference (add ?persona=ciso to filter), and see it in the frontend at /journey. Full narrative: examples/reference-project/README.md · persona catalog: docs/personas.md.
